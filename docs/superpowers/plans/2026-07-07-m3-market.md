# M3 Market (Auction House) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The auction house (SPEC-M3 step 6 / US5) — `GET /market/listings?itemId=&cursor=`, `POST /market/listings { itemId, qty, unitPrice }` (removes items from the inventory, txn), `POST /market/listings/:id/buy { qty }` (txn: 5 % tax → sink, transfer écus, item → buyer, partial buy decrements the stack), `DELETE /market/listings/:id` (seller only, returns the items).

**Architecture:** No realtime for the market (the SPEC-M3 §WS channel table lists no market event; the web refetches after a mutation). The tax rounding is a pure `marketTax` in `shared/formulas/economy.ts` (tested). All three mutations run in one Drizzle transaction with the listing row locked `for('update')`. Inventory debit/credit reuses a new generic `removeSellableQty` + the existing `addItem` (capacity-aware → `INVENTORY_FULL`).

**Tech Stack:** Drizzle, Zod, Fastify, Vitest. Reuses `toCharacterDto`, `regenContextFor`, `requireCharacter`, `addItem`, `inventoryCapacity`.

## Global Constraints

- Le serveur fait autorité ; **transaction** autour de tout mouvement d'écus + inventaire + annonce.
- Taxe **5 %** prélevée sur le paiement (sink) : l'acheteur paie `qty × unitPrice` (brut), le vendeur reçoit `brut − taxe`, la taxe disparaît. `marketTax` pure et testée (rule 7).
- Erreurs : `INSUFFICIENT_MATERIALS` (mise en vente sans les objets), `INSUFFICIENT_FUNDS`, `CANNOT_BUY_OWN_LISTING`, `LISTING_UNAVAILABLE` (annonce disparue / qty insuffisante), `INVENTORY_FULL` (sac plein à l'achat / annulation), `FORBIDDEN` (403, annulation par un tiers). Codes + messages FR déjà présents (`content/fr/errors.ts`).
- Achat partiel : décrémente `qty` de l'annonce ; annonce supprimée quand elle tombe à 0.
- **Décision de périmètre (tranchée) :** tout objet **non équipé** est vendable (matériaux, consommables, armes, armures). Le `listingSchema` figé ne porte pas la durabilité → une pièce non empilable revient à durabilité par défaut (null → traitée pleine) à l'achat/annulation. Simplification assumée, hors chemin critique Q5, documentée ici.
- Aucune chaîne FR en dur ; Zod en entrée ET sortie (rule 4).
- Vérif : `pnpm --filter @aldenfer/shared test`, `pnpm --filter api test`, builds `tsc --noEmit`.
- Hors périmètre : échange direct joueur↔joueur, PNJ marchand, prix planchers / quotas anti-abus (questions ouvertes M4).

- [ ] **Step 1: `removeSellableQty` (write availability-then-debit; no separate unit test — exercised by API tests)**
```ts
import { desc } from 'drizzle-orm';
export async function removeSellableQty(
  tx: Tx, characterId: string, itemId: string, qty: number,
): Promise<boolean> {
  const rows = await tx
    .select()
    .from(inventory)
    .where(and(eq(inventory.characterId, characterId), eq(inventory.itemId, itemId), eq(inventory.equipped, false)));
  const total = rows.reduce((s, r) => s + r.qty, 0);
  if (total < qty) return false;
  let need = qty;
  for (const row of rows) {
    if (need <= 0) break;
    const take = Math.min(need, row.qty);
    if (take === row.qty) await tx.delete(inventory).where(eq(inventory.id, row.id));
    else await tx.update(inventory).set({ qty: row.qty - take }).where(eq(inventory.id, row.id));
    need -= take;
  }
  return true;
}
```
