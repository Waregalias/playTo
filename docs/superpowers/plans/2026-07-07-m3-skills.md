# M3 Skills (Learn / Equip + wired effects) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax. Write tests first (TDD) for every formula and every wired effect.

**Goal:** SPEC-M3 step 7 / US6 — `POST /characters/me/skills { skillId }` (1 skill point + ember fragments for tiers 4–5, prerequisite = tier N−1 of the same branch, own class) and `PUT /characters/me/skills/equipped { slot1?, slot2? }` (≤2 active slots, learned + `kind:'active'` + `wiredInM3`; `null` unequips). Then wire the **câblé** effects into the systems that exist in M3: combat (equipped actives + passives), search loot, movement timer, vision, death loss. Contribution (Offrande) is already wired (step 5).

**Architecture:** Learn/equip eligibility is pure & tested (`skillLearnCheck`, `skillEquipCheck` in `shared/formulas/skills.ts`); services just apply the deltas in a transaction. Passive effects come from the existing `deriveSkillModifiers(learnedSkills)` (already tested). Each service computes `mods` once and applies the relevant field. A new pure `hexesInRange` (hex.ts) generalises discovery to `1 + visionBonus`.

**Tech Stack:** Drizzle, Zod, Fastify, Vitest. Reuses `deriveSkillModifiers`, `SKILLS_BY_ID`, `toCharacterDto`, `equippedGear`, `resolveAttack`, `applyXp`.

## Global Constraints

- Le serveur fait autorité ; **transaction** autour de tout apprentissage (débit point/fragments) et équipement.
- Zod entrée + sortie ; réponses = `characterSchema` (état complet, source de vérité).
- Erreurs : `SKILL_ALREADY_LEARNED`, `REQUIREMENT_NOT_MET` (prérequis / points / fragments / classe / non-apprise / passif à l'équipement). Codes + FR déjà présents.
- **Décision de périmètre (tranchée) : saignement (`bleedTurns`) différé.** Les actifs à saignement (Flèche barbelée `scout.hunt.2`, Litanie `cantor.verse.2`) appliquent leur coup de base ce jalon ; le DoT est noté « à câbler » (pas de nouvel état/migration combat).
- Effets `dodgePct` : **aucune compétence du catalogue ne le fixe** → non appliqué (rien à câbler). `fleeNoPenalty` (Évasion) → la fuite réussit toujours. Effets inertes (`wiredInM3:false`) restent apprenables mais non équipables.
- Simplicité : pas de refonte des formules de combat (`resolveAttack` non touchée) ; les modificateurs s'appliquent aux **entrées** (mitigation, multiplicateur, montants).
- Vérif : `pnpm --filter @aldenfer/shared test`, `pnpm --filter api test`, builds `tsc --noEmit`.

```ts
import type { CharacterClass } from '../constants/classes.js';
// …
export function skillLearnCheck(skillId, charClass, learned, skillPoints, emberFragments) {
  const skill = SKILLS_BY_ID[skillId];
  if (!skill || skill.class !== charClass) return { ok: false, code: 'REQUIREMENT_NOT_MET' };
  if (learned.includes(skillId)) return { ok: false, code: 'SKILL_ALREADY_LEARNED' };
  if (skill.tier > 1 && !learned.includes(`${skill.class}.${skill.branch}.${skill.tier - 1}`))
    return { ok: false, code: 'REQUIREMENT_NOT_MET' };
  if (skillPoints < 1 || emberFragments < skill.fragmentCost)
    return { ok: false, code: 'REQUIREMENT_NOT_MET' };
  return { ok: true };
}
export function skillEquipCheck(skillId, learned) {
  const skill = SKILLS_BY_ID[skillId];
  return !!skill && learned.includes(skillId) && skill.kind === 'active' && skill.wiredInM3;
}
```

(`hex.ts`) axial-range double loop as documented in hex algorithms; radius 1 must reproduce `center + neighbours`.
